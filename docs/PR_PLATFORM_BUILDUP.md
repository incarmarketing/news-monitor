# PR Platform Buildup

이 문서는 `news-monitor`를 단순 뉴스 모니터링 페이지가 아니라 인카금융서비스 PR 운영센터로 키우기 위한 제품 기준서입니다.

참고한 외부 구조:

- https://www.mediabee.com/pr/monitoring
- https://www.mediabee.com/pr/database
- https://www.mediabee.com/pr/distribution
- https://www.mediabee.com/pr/activity
- https://www.mediabee.com/pr/ai-assistant

## 핵심 판단

미디어비의 강점은 화면 디자인 자체보다 업무 묶음입니다. 뉴스 모니터링, 논조 분석, 알림, 자동 보고서, 기자/매체 DB, 보도자료 배포, 활동 이력, AI 작성 지원을 하나의 PR 업무 흐름으로 연결합니다.

우리 툴도 이제 기능을 하나씩 붙이는 방식에서 벗어나야 합니다. 방향은 "인카 PR Control Center"입니다.

## 제품 포지션

### 미디어비형 범용 PR 플랫폼

- 여러 기업이 쓰는 범용 기자 DB와 모니터링 플랫폼
- 넓은 매체 커버리지, 기자 검색, 배포, 협업, AI 작성까지 제공
- 장점: 기능 범위가 넓고 PR 업무 전체를 설명하기 좋음
- 한계: 우리 회사의 리스크 기준, 보험/GA 문맥, 내부 보고 형식에는 별도 커스터마이징 필요

### 인카 PR Control Center

- 인카금융서비스 홍보팀 전용 운영 도구
- 당사, 경쟁사, GA, 보험사, 정책/감독, 업계 동향을 회사 기준으로 분류
- 부정기사 24시간 감시, 카카오 알림, 내부 한장 보고서, 언론사 관리까지 내부 업무 흐름에 맞춤
- 목표: 범용 플랫폼보다 좁지만, 우리 팀 실무 판단은 더 빠르고 정확하게

## 제품 기둥

### 1. Situation Room

통합 대시보드의 첫 화면입니다. 사용자는 이 화면만 봐도 오늘의 PR 상황을 판단할 수 있어야 합니다.

- 오늘/최근 수집 기사
- 당사 직접 언급
- 당사 부정 기사
- GA/보험사/정책 동향
- 부정기사 감시 상태
- 카카오 알림 발송 이력
- 최신 핵심 이슈

원칙: 카드 수는 적게, 문장은 짧게, 판단에 필요한 수치와 상태만 남깁니다.

### 2. Monitoring Intelligence

뉴스 리스트가 아니라 분석 엔진이어야 합니다.

- 키워드 그룹: 당사, 경쟁사, GA, 보험사, 정책/감독, 업계 동향
- 논조 세분화: 긍정, 중립, 부정, 리스크, 해명 필요, 확산 우려
- 유사 기사 묶음: 같은 이슈를 중복 기사로 보지 않도록 클러스터링
- 매체 영향력: 언론사 중요도, 포털 제휴 여부, 과거 당사 보도 이력
- 24시간 부정기사 감시: 최근 10분 검사, 10분 주기 수행, 실패/지연 상태 표시

### 3. Report Studio

보고서는 화면 캡처가 아니라 산출물입니다.

- 일일, 주간, 월간 한장 보고서
- A4 한 장 기준의 인쇄/PDF 스타일
- 핵심 수치, 동향 분석, 근거 기사, 대응 필요 이슈만 남김
- 보고서별 자동 요약 문장
- 보고서 생성 이력과 공유 링크

### 4. Risk Response Desk

부정기사 발견 이후의 대응을 관리합니다.

- 기사 URL 또는 카드 선택
- 핵심 사실관계 자동 추출
- 사내 해명용 / 언론 해명용 / 임원 보고용 초안 생성
- 생성 전 확인 모달
- 버전 저장, 복사, 카카오/메일 공유
- 담당자와 처리 상태 관리

### 5. Media Relationship CRM

언론사 관리는 명단이 아니라 관계 이력입니다.

- 당사 보도자료를 한 번이라도 낸 언론사 자동 등록
- 언론사별 당사 기사, 부정 기사, 광고비, 접촉 이력
- 기자/담당자 메모, 통화, 미팅, 요청 사항
- 언론사 중요도와 관계 상태
- 더보기/필터/숨김 처리

### 6. Distribution & Performance

보도자료 배포와 성과를 한 흐름으로 묶습니다.

- 보도자료 캠페인 등록
- 타깃 언론사/기자 리스트
- 발송 이력, 성공 여부, 클릭/열람 기록
- 후속 보도 기사 자동 매칭
- 광고비 관리와 실제 보도 성과 연결

### 7. AI Workbench

AI 기능은 버튼 하나가 아니라 PR 실무 도구 묶음이어야 합니다.

- 보도자료 초안
- 기사 아이디어
- 해명자료 초안
- 임원 보고 요약
- 기자 피칭 메일
- FAQ와 Q&A
- 기존 생성 결과 저장/재생성/비교

### 8. Operations & Trust

자동화는 사용자가 믿을 수 있어야 합니다.

- GitHub Actions 최신 실행 상태
- 외부 cron 동기화 상태
- Supabase 저장 성공 여부
- 카카오 알림 발송 성공/실패
- 데이터 소스와 마지막 갱신 시각
- 로컬 저장 항목 점검

## 정보 구조 개편안

현재 화면을 아래 기준으로 재정렬합니다.

- 통합 상황판: 오늘의 상태, 부정기사 감시, 알림, 핵심 이슈
- 실시간 모니터링: 기사 검색, 필터, 스크랩
- 미디어 분석: 트렌드, 논조, 키워드, 경쟁사/업계 비교
- 리스크 대응센터: URL 분석, 사실관계, 초안 생성, 대응 상태
- 언론 CRM: 언론사/기자/활동 이력
- 배포/성과: 보도자료 캠페인, 발송, 클릭, 후속 보도
- 광고비 관리: 집행 내역, 월별/매체별 분석
- 보고서: 일일/주간/월간 한장 보고서
- 환경설정: 키워드 그룹, 사용자, 운영 상태

## 단계별 로드맵

### Phase 1. 신뢰도와 구조 안정화

- 부정기사 감시 24시간 10분 수행 신뢰도 고정
- 대시보드 카드 문구와 간격 정리
- 13시/18시 슬롯 데이터 보존 구조 점검
- 로컬 저장 항목 제거 또는 Supabase 이관
- README와 운영 문서 정리

### Phase 2. 분석 품질 고도화

- 키워드 그룹별 관리 UI 완성
- 부정 논조 세분화 기준 추가
- 유사 기사 묶음
- 매체 영향력/중요도 점수 도입
- 당사 직접 영향과 업계 일반 이슈 분리

### Phase 3. 보고서와 리스크 대응 완성

- 일일/주간/월간 한장 보고서 디자인 고정
- 리스크 대응 초안 생성 품질 개선
- URL 기반 사실관계 자동 채움
- 초안 버전 저장과 복사/공유
- 카카오 알림 상세 팝업과 발송 이력

### Phase 4. 언론 CRM과 배포 성과

- 당사 보도 언론사 자동 등록
- 기자/언론사 접촉 이력
- 보도자료 캠페인 관리
- 발송 성공, 클릭, 후속 기사 매칭
- 광고비와 보도 성과 연결

### Phase 5. PR AI Workbench

- 보도자료, 해명자료, 피칭 메일, 임원 보고 템플릿
- 생성 결과 저장/재생성/비교
- 사내 문체와 승인 흐름 반영
- 반복 업무 자동화

## 바로 다음 작업

1. 대시보드 메뉴와 화면명을 제품 구조에 맞춰 정리합니다.
2. 통합 상황판을 더 컴팩트하게 만들고, 핵심 카드만 첫 화면에 남깁니다.
3. 미디어 분석 화면을 "분석 리포트"가 아니라 실제 분석 도구처럼 개편합니다.
4. 언론 CRM은 언론사 목록, 당사 보도 이력, 접촉 메모를 한 화면에서 보게 만듭니다.
5. 보고서는 A4 한 장 산출물 기준으로 고정하고, 화면용 대시보드와 분리합니다.

## 설계 원칙

- 사용자는 홍보팀 실무자입니다. 멋진 랜딩페이지보다 빠른 판단, 반복 업무, 보고 산출물이 중요합니다.
- 숫자는 많게, 설명은 짧게 둡니다.
- 부정기사와 리스크는 항상 첫 화면에서 상태가 보여야 합니다.
- 보고서는 무조건 한 장으로 끝나는 구조를 우선합니다.
- 데이터는 로컬이 아니라 Supabase와 GitHub Actions 중심으로 관리합니다.
- 미디어비를 따라 하는 것이 아니라, 우리 회사 업무 기준을 더 깊게 반영합니다.
